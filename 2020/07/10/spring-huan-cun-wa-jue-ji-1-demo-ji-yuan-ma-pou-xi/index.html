Spring缓存挖掘机（一）- demo及源码剖析

Spring cache Caffeine
Spring缓存组件

背景

由于系统中存在多方请求设备服务获取设备运行状态的现象，易出现重复请求的问题，故需要使用缓存去减缓调取频率，具体的方案是多级缓存操作
本地缓存+redis缓存+调设备服务（慢，耗资源）
先查本地缓存，若无则查redis缓存，否则再调服务


缓存方案

先看下市面上现有的缓存框架有哪些



Spring 内置缓存组件


先阅读下官方文档：https://docs.spring.io/spring-boot/docs/current/reference/html/spring-boot-features.html#boot-features-caching

Spring缓存管理器

阅读文档可知，Spring提供了中央缓存管理器的底层SPI接口：org.springframework.cache.CacheManager，，在spring-context 模块内。
基于它可由各种第三方的缓存管理器去实现和扩展，目前Spring 同样内置了以下几种第三方缓存组件管理器，命名规范为***Manager


Spring缓存

Spring 也提供了缓存通用操作接口org.springframework.cache.Cache，在spring-context 模块内
而在Cache接口下，Spring内置了几种市面上常见缓存方案的实现，命名规范为***Cache

以上，RedisCache位于spring-data-redis，其他的都在spring-context-support中。
注意，可以看到，在Spring5 中 已经没了GuavaCache的身影，取而代之的是Caffeine。

缓存 说明
CaffeineCache 是通过JAVA8对Guava缓存的重写版本，（在Spring5）取代了对Guava缓存的支持，在5.2.4中要求Caffeine版本为2.1或以上
ConcurrentMapCache 基于ConcurrentMap实现的缓存技术，无过期时间，需手动删除
JCacheCache 基于JCache标准的实现作为缓存技术
RedisCache 基于Redis实现的缓存技术
EhCacheCache 基于EhCache实现的缓存技术
NoOpCache 不会实际存储缓存，适用于禁用缓存的无操作实现

而在Springboot中，通过org.springframework.boot.autoconfigure.cache.CacheType可以发现，Springboot还支持了其他类型的缓存框架

这里也大概说明一下

缓存 说明
HAZELCAST 全称是Hazelcast IMDG (In-Memory Data Grid)，内存数据网格
是一个开源的分布式内存数据存储和计算平台
提供多种分布式数据结构如Map/Queue/Topic等
INFINISPAN 是一款基于Apache v2.0版本的开源数据网格系统
COUCHBASE 是一款非关系型数据库，MemBase（存储键值对）与couchDb（存储json文档）这两个NoSQL数据库合并的产物，

注解

增加了对应的注解方式以支持缓存功能：@EnableCaching、@CacheConfig、@Cacheable、@CachePut、@CacheEvict等

@EnableCaching

启用Springboot 的缓存管理功能，等同于<cache: * > 的xml文件配置。当开启时，CacheManager的bean必须指定手动注册，目前暂未有合理的缓存框架可以用作惯例。
当要将 @EnableCaching与CacheManager等建立合理的联系时，Spring已经提供了接口CachingConfigurer用于实现

属性 说明
proxyTargetClass 是否要基于cglib的代理方式，缺省为false，即基于jdk动态代理
mode 代理实现方式（动态代理PROXY，静态代理ASPECTJ），缺省为动态代理

@CacheConfig

类级别的注解。用于提供一种在类级别共享与缓存相关的配置。当注解在类上时，会提供基于该类的默认配置，即可能被覆盖

属性 说明
cacheNames 缓存操作对应的名称，可多个
keyGenerator 要使用的自定义key生成器的bean名称，当缓存操作未设置任何值时，使用它替代默认值
cacheManager 要使用的自定义cacheManager的bean名称，当缓存操作未设置解析器和管理器时，且没通过cacheResolver属性设置解析器时，使用它替代默认值
cacheResolver 要使用的自定义cacheResolver的bena名称，当缓存操作未设置解析器和管理器时，使用它替代默认值

@Cacheable

被注解的方法，支持在有缓存时直接返回，无缓存（或缓存过期）时执行方法，并在方法返回时指定缓存名称和key缓存最新数据
适用场景：注解方法需要做缓存以 缓解调用频率对底层service/db的影响，通过过期时间可灵活设置数据更新

属性 说明
cacheNames 缓存操作对应的名称，可多个
value 同上
key 默认使用方法参数作为key值，也可以借助SpEL表达式组装key值
keyGenerator 与key功能互斥，要使用的自定义cacheManager的bean名称，可以实现org.springframework.cache.interceptor.KeyGenerator使用
cacheManager 要使用的自定义cacheManager的bean名称
cacheResolver 与cacheManager互斥，要使用的自定义cacheResolver的bena名称
condition 缓存条件，使用SpEL表达式来实现，只有条件成立才会缓存
unless 否决缓存条件，使用SpEL表达式来实现，即在方法调用后执行该条件，只有条件不成立才会缓存
sync 是否开启同步调用，默认false，当为true时多线程执行时则会同步调用注解方法（注意，前提需要第三方缓存框架提供同步支持，自定义实现时也要考虑）

@CachePut

与@Cacheable相反，它会一直执行注解方法而不会跳过，并每次将结果存储在缓存中，底层触发Cache的put()
适用场景：注解方法需要被调用以及时更新缓存

它的属性和@Cacheable 一样，只不过少了sync，因为它不需考虑是否要同步调用

@CacheEvict

被注解的方法，支持在调用方法前/后 执行缓存逐出操作，底层触发Cache的evict()
适用场景：注解方法需要在特定时间被调用以逐出缓存，例如一个设备启动并结束后，关乎它的某些状态已无需维护，故在结束那一刻逐出缓存

它的属性和@Cacheable也差不多，下面看些不一样的属性

属性 说明
allEntries 是否逐出指定缓存名称下的所有key，缺省为false，仅逐出指定key的缓存；true则在方法执行成功后逐出所有key的缓存
beforeInvocation 是否在调用方法之前逐出指定key缓存，缺省为false，当方法执行成功后才会逐出缓存；true则无需关心方法的调用结果，直接逐出

示例demo

在分析源码之前，先使用一个demo来分析（注意，这里要引入caffeine的依赖）

  • 先定义了一个缓存的配置类CustomCacheConfig，使用@EnableCaching开启缓存支持，定义缓存名称，初始化缓存管理器CaffeineCacheManager，带入缓存名称。
@EnableCaching
@Configuration
public class SaasCacheConfig {

    /**
     * 全局缓存名称
     */
    public static final String cacheName = "saas-cache";

    @Bean(name = {"cacheManager"})
    public CacheManager getCache() {
        return new CaffeineCacheManager(cacheName);
    }
}
  • 编写一个service接口和实现类，在其接口方法上添加缓存注解，sync代表同步访问
public interface DemoService {

    String getData(String s1, String s2, Integer i1);

    void clearData(String s1, String s2, Integer i1);
}
@Service
public class DemoServiceImpl implements DemoService {
    @Cacheable(value = SaasCacheConfig.cacheName, key = "#s1.concat(#s2)", sync = true)
    @Override
    public String getData(String s1, String s2, Integer i1) {
        System.out.println("none cache, start to save...");
        return "success";
    }
    @CacheEvict(value = SaasCacheConfig.cacheName, key = "#s1.concat(#s2)", beforeInvocation = true)
    @Override
    public void clearData(String s1, String s2, Integer i1) {
        // empty method
    }
}

Spring提供了org.springframework.cache.interceptor.CacheInterceptor和org.springframework.cache.interceptor.CacheAspectSupport用于声明式缓存

CacheAspectSupport

该类包含了与Spring基础的缓存api的集成功能。

下面分析下源码

拦截器进入到execute()

每次进行切面，都会执行execute()

protected Object execute(CacheOperationInvoker invoker, Object target, Method method, Object[] args) {
    // Check whether aspect is enabled (to cope with cases where the AJ is pulled in automatically)
    if (this.initialized) {
        // 获取被代理类
        Class<?> targetClass = getTargetClass(target);
        // 获取可操作的缓存资源并指定被代理类+方法进行过滤，得到一个可操作集合
        CacheOperationSource cacheOperationSource = getCacheOperationSource();
        if (cacheOperationSource != null) {
            Collection<CacheOperation> operations = cacheOperationSource.getCacheOperations(method, targetClass);
            if (!CollectionUtils.isEmpty(operations)) {
                return execute(invoker, method,
                        new CacheOperationContexts(operations, method, args, target, targetClass));
            }
        }
    }

    return invoker.invoke();
}

上面得到的缓存操作对象CacheOperation 如下

Builder[public java.lang.String source.cache.service.DemoServiceImpl.getData(java.lang.String,java.lang.String,java.lang.Integer)] caches=[saas-cache] | key='#s1.concat(#s2)' | keyGenerator='' | cacheManager='' | cacheResolver='' | condition='' | unless='' | sync='true'

构造缓存操作上下文

public CacheOperationContexts(Collection<? extends CacheOperation> operations, Method method,
        Object[] args, Object target, Class<?> targetClass) {

    this.contexts = new LinkedMultiValueMap<>(operations.size());
    for (CacheOperation op : operations) {
        this.contexts.add(op.getClass(), getOperationContext(op, method, args, target, targetClass));
    }
    this.sync = determineSyncFlag(method);
}
public CacheOperationContext(CacheOperationMetadata metadata, Object[] args, Object target) {
    this.metadata = metadata;
    this.args = extractArgs(metadata.method, args);
    this.target = target;
    this.caches = CacheAspectSupport.this.getCaches(this, metadata.cacheResolver);
    this.cacheNames = createCacheNames(this.caches);
}

创建缓存实例

// org.springframework.cache.interceptor.CacheAspectSupport#getCaches
protected Collection<? extends Cache> getCaches(
        CacheOperationInvocationContext<CacheOperation> context, CacheResolver cacheResolver) {

    Collection<? extends Cache> caches = cacheResolver.resolveCaches(context);
    if (caches.isEmpty()) {
        throw new IllegalStateException("No cache could be resolved for '" +
                context.getOperation() + "' using resolver '" + cacheResolver +
                "'. At least one cache should be provided per cache operation.");
    }
    return caches;
}
// org.springframework.cache.interceptor.AbstractCacheResolver#resolveCaches
public Collection<? extends Cache> resolveCaches(CacheOperationInvocationContext<?> context) {
    Collection<String> cacheNames = getCacheNames(context);
    if (cacheNames == null) {
        return Collections.emptyList();
    }
    Collection<Cache> result = new ArrayList<>(cacheNames.size());
    for (String cacheName : cacheNames) {
        // 获取当前设置的缓存管理器中指定名称的缓存实例
        Cache cache = getCacheManager().getCache(cacheName);
        if (cache == null) {
            throw new IllegalArgumentException("Cannot find cache named '" +
                    cacheName + "' for " + context.getOperation());
        }
        result.add(cache);
    }
    return result;
}

若开启了同步模式

private Object execute(final CacheOperationInvoker invoker, Method method, CacheOperationContexts contexts) {
    // 如果开启了同步（即sync=true），则进入if
    if (contexts.isSynchronized()) {
        CacheOperationContext context = contexts.get(CacheableOperation.class).iterator().next();
        // 当注解上添加条件condition时，这里需要校验是否通过
        if (isConditionPassing(context, CacheOperationExpressionEvaluator.NO_RESULT)) {
            // 获取key值
            Object key = generateKey(context, CacheOperationExpressionEvaluator.NO_RESULT);
            // 得到具体的缓存对象，基于SPI机制，这里可以出现不同类型的缓存对象，如CaffeineCache
            Cache cache = context.getCaches().iterator().next();
            try {
                // 如何同步处理的核心就在cache.get这里，不同的缓存组件已经对应封装了同步逻辑，如果缓存存在则直接返回，不存在则同步获取资源
                // 
                return wrapCacheValue(method, cache.get(key, () -> unwrapReturnValue(invokeOperation(invoker))));
            }
            catch (Cache.ValueRetrievalException ex) {
                // The invoker wraps any Throwable in a ThrowableWrapper instance so we
                // can just make sure that one bubbles up the stack.
                throw (CacheOperationInvoker.ThrowableWrapper) ex.getCause();
            }
        }
        else {
            // 条件不通过，无需缓存直接调用基础方法
            return invokeOperation(invoker);
        }
    }
}

若是异步模式（默认）

清除代码如下

protected void doEvict(Cache cache, Object key, boolean immediate) {
    try {
        if (immediate) {
            cache.evictIfPresent(key);
        }
        else {
            cache.evict(key);
        }
    }
    catch (RuntimeException ex) {
        getErrorHandler().handleCacheEvictError(ex, cache, key);
    }
}

清除主要调用了缓存实现对应的evictIfPresent或evict方法

缓存主逻辑
private Object execute(final CacheOperationInvoker invoker, Method method, CacheOperationContexts contexts) {
    // 校验上下文是否命中了缓存操作对象CacheableOperation，如果当前是清除操作则不命中
    Cache.ValueWrapper cacheHit = findCachedItem(contexts.get(CacheableOperation.class));

    // 如果找不到则收集未命中的缓存
    List<CachePutRequest> cachePutRequests = new LinkedList<>();
    if (cacheHit == null) {
        collectPutRequests(contexts.get(CacheableOperation.class),
                CacheOperationExpressionEvaluator.NO_RESULT, cachePutRequests);
    }

    Object cacheValue;
    Object returnValue;
    // 如果命中缓存，并且没有收集缓存，则获取缓存数据
    if (cacheHit != null && !hasCachePut(contexts)) {
        cacheValue = cacheHit.get();
        returnValue = wrapCacheValue(method, cacheValue);
    }
    else {
        returnValue = invokeOperation(invoker);
        cacheValue = unwrapReturnValue(returnValue);
    }

    // 收集任何显式的 @CachePuts
    collectPutRequests(contexts.get(CachePutOperation.class), cacheValue, cachePutRequests);

    // Process any collected put requests, either from @CachePut or a @Cacheable miss
    for (CachePutRequest cachePutRequest : cachePutRequests) {
        cachePutRequest.apply(cacheValue);
    }
    return returnValue;
}
执行后清除缓存
processCacheEvicts(contexts.get(CacheEvictOperation.class), false, cacheValue);
